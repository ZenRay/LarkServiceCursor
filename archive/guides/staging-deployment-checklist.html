🆘 回滚流程

如果发现重大问题需要回滚:

1. 应用回滚

# 切换到上一个稳定版本
cd /opt/lark-service
git checkout <previous-stable-tag>

# 重启应用 (如果是独立服务)
systemctl restart lark-service

2. 数据库回滚

# 参考: docs/database-migration-rollback.md
alembic downgrade <target-version>

3. 配置回滚

# 恢复上一个配置文件
cp .env.staging.backup .env.staging

4. 验证回滚

  • 验证应用功能正常

  • 验证数据一致性

  • 通知相关团队
